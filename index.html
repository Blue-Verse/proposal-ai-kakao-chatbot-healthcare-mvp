제안 포인트

1. 헬스케어 + OpenAI 분석 실서비스 운영 경험

시니어 헬스케어 SaaS Harmony Link에서 OpenAI 기반 건강 분석, 케어 로그·투약·알림 시스템을 직접 설계·운영한 경험을 보유하고 있습니다. 본 프로젝트의 "건강 데이터 → GPT 분석 → 맞춤형 응답" 흐름을 즉시 구현 가능한 노하우를 그대로 적용합니다.

2. 카카오톡 채널 챗봇 아키텍처 설계 역량

카카오 i 오픈빌더 스킬서버 방식과 Webhook 직접 호출 방식의 트레이드오프(상태관리, 응답 지연, 운영 편의성)를 기획 단계에서 비교 검토하여 클라이언트 요구사항에 최적화된 구조를 제안합니다. 응답 5초 제한 대응을 위한 Callback API 활용 방안까지 사전 설계합니다.

3. 다중 외부 API 안정 연동 노하우

P2P Funding 프로젝트에서 NICE 본인인증(CheckPlus, SafeKey, NPAC) + 가상계좌 + 다수 금융 API를 동시 안정 연동한 경험, Series-B에서 ChatGPT + KYC + 다수 규제 API를 단일 백엔드에서 통합 운용한 경험을 본 프로젝트에 그대로 적용합니다. 공공 API의 일시적 장애나 응답 지연에 대한 재시도·캐싱·폴백 정책을 표준 패턴으로 적용합니다.

4. 60일 일정 준수 (1순위 부합)

유사 풀스택 + 외부 API + AI 챗봇 프로젝트를 60일 내 MVP 단계로 출시한 경험을 기준으로 본 프로젝트의 일정을 산정했습니다. Phase별 마일스톤 검수와 주 1회 정기 미팅으로 일정 리스크를 사전 차단합니다.

5. 정부지원사업 결과 보고서 대응

요구사항 정의서, 화면설계서, IA, API 연동 명세서, 결과 산출물 패키지를 정부지원사업 결과 보고에 활용 가능한 형식으로 일관되게 정리하여 납품합니다.

6. 월 단위 유지보수 및 2·3차 고도화 연속 협업

MVP 출시 이후 월 단위 유지보수, 2차/3차 고도화까지 동일한 팀이 연속 수행할 수 있습니다. 코드베이스 인수인계 비용 없이 빠른 기능 추가가 가능합니다.

PRD (Product Requirements Document)

NDA 체결 후 공유될 상세 정의서(특허 로직, 화면설계서, API 연동 명세서)에 따라 본 PRD는 확정·보완됩니다. 본 문서는 공고 본문 기준 1차 안입니다.

1. 프로젝트 배경 및 목적

개인의 건강 상태와 라이프스타일에 맞춘 헬스케어 서비스를 카카오톡 채널이라는 친숙한 인터페이스로 제공하는 것이 본 프로젝트의 핵심 목표입니다. 사용자는 별도 앱 설치 없이 카카오톡에서 대화를 주고받는 것만으로 공공·민간 건강데이터 기반 개인 맞춤형 분석을 받고, 정기 알림톡으로 행동 유도까지 이어지는 경험을 얻게 됩니다. MVP 단계에서는 복잡한 기능보다 핵심 가설(카카오톡 인터페이스 + AI 분석 + 자동 알림) 검증과 빠른 출시를 우선합니다.

2. 사용자 정의

사용자 유형설명주요 기능
